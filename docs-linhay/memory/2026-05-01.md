## Session Management 浏览器态桥接修复
- `http://localhost:34115/#frame=session-management` 的空白页和 `Failed to fetch` 问题，本轮确认不是会话解析本身错误，而是浏览器态桥接判定过宽：页面存在 `window.runtime` 时被误认为可直接调用 `window.go.main.App`，导致 `GetVersion()` 等 Wails 包装函数在无 `go.main.App` 的浏览器页里崩溃。
- 已新增 `hasWailsAppBindings()`，把“仅有 runtime”与“runtime + go.main.App 可用”区分开；`App.tsx` 和 `UsageDeskWorkspace.tsx` 改为只在完整 Wails 绑定存在时才走 Go API / 事件订阅。
- `session-management` 入口同时收紧为“有本地快照先显示，不自动重刷”，避免 dev bridge 暂不可用时把已有页面态打成错误页；dev bridge 单次超时也从 `800ms` 放宽到 `5000ms`，降低首扫误判失败。
- 本轮在浏览器直接复测 `http://localhost:34115/#frame=session-management`，已能看到真实数据：`99 个项目 / 2861 条会话`，并能展开项目列表；当前这次会话里额外启动了一个本地 Vite bridge 在 `http://localhost:5174`，因为旧的 `5173` 被僵尸进程占用。
