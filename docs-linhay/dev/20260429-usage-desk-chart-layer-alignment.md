# Usage Desk 图表分层对齐与验收规则

## 背景

`UsageDesk` 的天级趋势同时使用了两套渲染层：

- `svg`：折线、面积、选中竖线
- HTML：点位、数值标签、底部日期标签、点击热区

这类混合渲染在时间窗口切换时很容易出现“线在一个坐标系里，点在另一个坐标系里”的错位。

## 规则

1. `svg` 与 HTML 点位层必须共用同一个逻辑宽度。
2. 不要让 `svg` 被容器拉伸到全宽，而点位仍按旧 `chartWidth` 计算。
3. `7D / 14D` 这类短窗口不需要为了填满卡片而强行拉宽，应优先保留按点数推导出的自然宽度。
4. 时间窗口切换的动画应该作用在曲线本身，而不是整块图表卡片：
   - 折线 sweep
   - 面积淡入
   - 点位上浮

## 验收步骤

1. 打开真实页面，而不是只看本地静态推断。
2. 在目标页面切换 `7D / 14D / 30D / 全部`。
3. 检查：
   - 点位是否压在线上
   - 短窗口宽度是否自然收窄
   - 曲线动画是否发生在曲线图层本身
4. 截图归档到对应 `space/screenshots/`。

## 本次会话结论

- `7D / 14D` 的点线错位根因是 `svg` 与 HTML 点位层宽度坐标系不一致。
- 最终裁定是：
  - 短窗口保留自然宽度
  - 曲线动画下沉到图层本身
  - 用真实页面截图作为最终验收标准
