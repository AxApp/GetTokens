设计备注

前端规划规则

  1. v1 入口仍在 `Accounts`，但新增独立的 `Usage Desk` 子工作区，而不是继续扩张卡片和模态。
  2. 页首只有三组控制：`source kind`、`provider`、`time range`。避免再叠一套账号过滤器把语义混掉。
  3. `provider` 必须是持久状态，不是临时 UI 装饰；即使 v1 只有 `Codex`，结构也不能写死。
  4. `ObservedRequestUsage` 与 `LocalProjectedUsage` 共用页面骨架，但不能共用文案语义和摘要口径。
  5. 账号卡片继续展示 `quota + recent health`，但不展示本地 token、session 或 local requestCount。
  6. 详情钻取从“账号详情模态”中剥离出来，改为 usage workspace 自己的右侧详情区或底部抽屉。
明确非目标

v1 不做本地 usage 到 `auth-file` 的归因，不做 local usage 写回账号卡片，不做跨 provider 本地 usage 总计。

建议路由
`/accounts` 保持一级入口，并新增 `usage desk` 子工作区；`/status` 继续是 relay/service 入口。
