### 2. Codex API key 卡片重复与删除修复
修复账号池 Codex API key 模式中“添加后出现两张卡片”和“最后重复卡片无法删除”的问题。

- 新建 space：`docs-linhay/spaces/20260502-codex-api-key-card-delete/README.md`。
- 决策：Codex API key 的前端展示/选择继续使用稳定 `local-id`，但合并、查重、删除、重命名、优先级和禁用操作都要同时接受稳定 ID 与配置派生 ID。
- 修复：`mergeCodexAPIKeyInputs` 同时按记录 ID 与 `apiKey + baseUrl + prefix` 配置身份去重，避免 sidecar 无 `local-id` 的镜像记录被展示为第二张卡。
- 修复：`DeleteCodexAPIKey` 等变更入口通过配置派生 ID 兜底匹配本地稳定记录，确保点到重复派生卡时也能删除真实本地记录并同步 sidecar。
- 验收：新增 `TestMergeCodexAPIKeyInputsDeduplicatesStoredStableIDAndSidecarMirror` 与 `TestDeleteCodexAPIKeyAcceptsDerivedConfigIDForStableLocalRecord`，并通过 `go test ./internal/wailsapp ./internal/accounts`、`go test ./...`。
