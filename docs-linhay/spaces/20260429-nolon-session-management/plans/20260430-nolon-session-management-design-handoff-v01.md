# Nolon Session Management Design Handoff

## 目的

本文件用于把当前 `design-preview.html` 的页面结构、交互边界和执行注意事项收口成一份可以直接交给后续实现者的交接文档。

当前文档最初用于设计交接；自 2026-04-30 起，第一页已完成首轮工程实现，并接入真实 rollout 数据。

## 当前唯一入口

- 设计稿入口：[design-preview.html](/Users/linhey/Desktop/linhay-open-sources/GetTokens/docs-linhay/spaces/20260429-nolon-session-management/design-preview.html)

## 页面信息架构

1. 左栏是 `项目列表`。
2. 右栏是当前项目对应的 `项目会话` 列表。
3. 主页面不常驻展开会话详情。
4. 点击某条会话后，弹出一个 `会话文件弹层`。

## 主页面裁定

### 左栏：项目列表
- 纵向列表，不做卡片墙。
- 每行显示：
  - 项目名
  - 会话数量
  - provider 概况
  - rewrite 候选概况
  - 最近活跃时间

### 右栏：项目会话
- 仍是纵向列表，不做两列卡片区。
- 列表头右侧固定为三个筛选项：
  - `全部`
  - `活跃`
  - `已归档`
- 每条会话行当前按“一个会话文件”理解，不在列表里展开消息详情。

## 弹层裁定

### 基本原则
- 弹层不是左右分栏工作台。
- 弹层主体是 `消息列表`。
- 一条消息一行。
- 其他辅助信息全部压缩到顶部摘要区。

### 顶部摘要区
- 只保留压缩信息，例如：
  - 总消息数
  - 当前会话文件数量
  - 是否脱敏
  - 当前选中消息
  - 角色分布
  - 主题摘要

### 主体区
- 只保留纵向消息列表。
- 每条消息一行，建议包含：
  - 消息序号
  - 角色
  - 时间或脱敏时间标记
  - 消息标题
  - 一行摘要

## 内容样本边界

1. 当前弹层里的消息样本来自本机真实 rollout 文件。
2. 当前稿件已经做脱敏处理：
   - 不展示绝对路径
   - 不展示会话 id / call id
   - 不展示源码定位
   - 不展示环境变量与机器上下文
3. 后续若替换样本，必须保持同样的脱敏边界。

## 不要回退的点

1. 不要再回到“右侧常驻详情”布局。
2. 不要再把弹层做成左右双栏。
3. 不要再把主体区做成解释模块堆叠。
4. 不要再用虚构对话占满弹层主体；优先保留真实样本的节奏感。
